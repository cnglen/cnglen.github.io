Content based RS

How to build?

  • compute vectors to describe item
  • building profiles of user preference
  • predicting user interest in items

item vector

  • simple 0/1 (keyword occurs?)
  • occurence count
  • TF-IDF
  • often normalized

user vector

  • user could build/edit own profile
  • infer profile from actions (implict)
  • infer profile from ratings (explict)

Strength and chanllenge

strength:

  • understandable
  • easy computation
  • new item

drawback:

  • figuring out the right weights and factors: limited content analysis
  • highly simplified model
  • new user
  • lack of serendipity

Neightborhood CF

User-user CF

  • \(\hat{r}_{ui} = \bar{r}_u + \frac{\sum_{v\in N_{i}(u)} w_{uv} (r_{vi} - \bar{r}_v)} {\sum_{v\in N_{i}(u)} |w_{uv}|}\)
  • user bias

Item-item CF

  • \(\hat{r}_{ui} = \bar{r}_i + \frac{\sum_{j\in N_{u}(i)} w_{ij} (r_{uj} - \bar{r}_j)} {\sum_{v\in N_{u}(i)} |w_{ij}|}\)
  • item bias

How to implement?

  • normalization of ratings (mean centering, z-score)
  • computation of similarity (wuv, wij)
    • cosine/pearson correlation
    • significance of a weight
      • wuv is penalized by a factor proportional to the # of commonly rated items
    • variacne of ratings
  • selection of neighbors
    • Top-N
    • threshold

Strength and drawback

  • limited coverage
  • sensitivity to sparse data

Matrix factorization Models

  • SVD
  • SVD++: implict feedback considered
  • TimeSVD++: timporal dynamics considered

Evaluation of RS

experimental settings

  • offline
  • user study
  • online

properties

  • user preference (用户满意度)
    • user study
      • user sample, importane of users
      • binary vs non-binary answers
      • subjective, need more users
  • prediction accuracy (offline)
    • RMSE/MAE (regression)
    • Precison/Recall/TPR/FPR @TopN (classification)
    • NDCG(ranking)
  • Coverage (覆盖率)
    • user space
      • proportion of users for which RS can recommend
      • coverage and accuracy trade-off
    • item space
      • longtail items
      • percentage of all items that can ever be recommendated
      • entropy
  • novelty(新颖性)
    • people not known
  • serendipity(惊喜度)
    • suprising
    • successful
    • Get people to sonsuming less popular items
  • diversity(多样性)
    • Don't turn away customers who are not currently intereted in a narrow portion of catalog
    • opposite of similarity @ Top N
  • robustness
    • fake information
    • attack
  • scalability (growth of data)
  • adaptivity (time)
